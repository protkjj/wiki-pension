## 절대 원칙 (5개) — DRAFT

계산 엔진이 어떤 상황에서도 반드시 지켜야 하는 원칙. 모든 코드/리뷰는 이 5개를 기준으로 점검한다.

1. **재현성(Reproducibility).** 동일 입력 + 동일 config → 항상 바이트 단위로 동일한 산출물.
   난수·실행시각·딕셔너리 순서 등 비결정적 요소가 결과값에 개입해서는 안 된다.
   (실행시각은 로그의 메타데이터로만 기록하고 계산에는 쓰지 않는다.)

2. **투명성·감사가능성(Auditability).** 모든 최종 수치는 개인별 연도별 상세 테이블의
   기여분 합으로 역추적 가능해야 한다. 블랙박스 중간값 금지. 임의의 사번에 대해
   손계산으로 재현할 수 있는 상세 덤프를 항상 제공한다.

3. **개인정보 보호(Privacy).** 성명 등 직접 식별정보는 받지도 저장하지도 않는다
   (사번만 사용). 실데이터가 들어가는 `data/`는 git에서 제외한다.

4. **설정 기반(Config-driven).** 할인율·임금상승률·정년·각종 convention 등 계산에
   영향을 주는 모든 가정은 YAML로 제어한다. 코드에 가정값을 하드코딩하지 않는다.
   기본값은 모델에 명시하고 문서화한다.

5. **검증 우선(Validation-first).** 모든 핵심 수식은 손계산으로 검증된 테스트를 갖는다.
   명부 입력은 계산 전에 검증하며, 오류(계산 중단)와 경고(플래그 후 진행)를 구분한다.

## 도메인 지식: K-IFRS 1019 PUC 방식 — DRAFT (프롬프트 2 착수 전 확정 필요)

> 아래 수식은 **초안**이다. timing convention(탈퇴시점·할인기간·임금상승 반영시점) 및
> CSC 산출 방식은 회사 관행에 따라 달라질 수 있으므로, 실제 엑셀 계산과 대사(프롬프트 4)
> 하기 전에 확정한다. 코드에서는 이 선택지들을 config로 노출한다.

### 기본 아이디어

예측단위적립방식(PUC)은 종업원이 정년까지 매년 근무함으로써 발생하는 미래 퇴직급여를
현재 시점의 근속에 비례해 배분하고, 각 미래 탈퇴시점에서 지급될 급여의 기대현재가치를
합산하여 확정급여채무(DBO)를 산정한다.

기호:
- `d0` = 산출기준일(valuation_date)
- `x` = 기준일 현재 도달연령, `s` = 기준일 현재 도달근속(중간정산자는 중간정산기준일부터 기산)
- `R` = 정년(종업원구분별). 정년까지 남은 연수 `n = R - x`
- `t` = 미래 연도 인덱스 (기준일 익일 ~ 정년, t = 1..n)
- `B(t)` = t시점 탈퇴 시 예상 퇴직급여
- `sal(t)` = t시점 예상 기준급여(월평균임금), 임금상승률 반영
- `S(t)` = t시점 탈퇴 시 총근속연수
- `m` = 적용배수(multiplier)
- `i` = 할인율(discount_rate), `v = 1/(1+i)` = 할인계수의 1년분

### 예상 퇴직급여

일시금 기준 예상 퇴직급여:

```
B(t) = sal(t) × S(t) × m
```

- `sal(t)` = base_salary × (1 + 임금상승률)^(임금상승 반영연수).
  반영연수는 `salary_increase_timing`(start/mid/end of year)에 따라 t 또는 t-1, t-0.5 등으로 정한다.
- `S(t)` = 탈퇴시점 총근속. `service_day_count`(act/365, act/365.25, months)에 따라 일할한다.

### 재직비율 (PUC 배분 계수)

현재까지 적립된 몫만 부채로 인식:

```
누적비율(t) = s / S(t)        # 현재근속 / 탈퇴시점 총근속
```

### 탈퇴 확률 (다중탈퇴)

각 미래 연도 t에 대해:
- `p_stay(t)` = 기준일부터 t-1년말(또는 timing에 따른 시점)까지 재직잔존확률
  = ∏ (1 − 퇴직률 − 사망률) 누적
- `q_ret(t)` = t년 중 퇴직으로 탈퇴할 당기 확률
- `q_death(t)` = t년 중 사망으로 탈퇴할 당기 확률
- `q_ret_age(n)` = 정년 도달 확률(마지막 연도)

퇴직률은 `retirement_rate_basis`(age|service)에 따라 도달연령 또는 도달근속으로 조회한다.
사망률은 성별·연령별 테이블에서 조회한다.
`decrement_timing`(end_of_year|mid_year)에 따라 탈퇴가 연중 언제 발생하는지 가정한다.

### 할인계수

탈퇴시점까지의 할인기간을 `discount_timing`으로 정한다(탈퇴 timing과 정합).
예: mid_year 탈퇴면 할인기간 ≈ t − 0.5.

```
disc(t) = v^(할인기간(t)) = (1+i)^(−할인기간(t))
```

커브(기간별 할인율)로 확장 가능하도록 `disc(t)`는 함수 형태로 둔다.

### DBO 기여분 / 개인 DBO

각 연도 t의 DBO 기여분 = (그 시점 탈퇴로 지급될 급여의 현재까지 적립분의 기대현재가치):

```
DBO_contrib(t) = B(t) × (s / S(t)) × (탈퇴확률: q_ret(t)/q_death(t)/정년도달) × disc(t)
개인 DBO = Σ_t DBO_contrib(t)
```

### CSC(당기근무원가) 기여분

당기근무원가는 당해 1년의 근무로 추가 적립되는 급여분의 기대현재가치.
`csc_method` convention 선택지(예: 총근속 대비 1년분 배분 방식 등)에 따라 산출식을 고른다.
초안:

```
CSC_contrib(t) = B(t) × (1 / S(t)) × (탈퇴확률) × disc(t)
개인 CSC = Σ_t CSC_contrib(t)
```

### 제도구분(plan_type) 처리

- `1` = DB 정상평가: 위 PUC 수식으로 계산.
- `2` = 간편법: 당년도추계액(current_year_accrual)을 그대로 부채로 계상.
- `3` = 제외: 결과에서 제외하되 별도 목록으로 출력.

## 기술 규약

- 언어/런타임: Python 3.11+
- 데이터 모델: pydantic v2
- 수치: numpy(내부 연도 루프 벡터화), pandas(명부·산출표)
- 설정: YAML
- 산출물: xlsx(openpyxl), 실행로그 JSON
- 테스트: pytest
- 패키지 레이아웃: `src/dbo` (src layout), CLI 진입점 `dbo`
- 통화 단위: 원(KRW). 최종 반올림 단위는 `rounding`(기본 1원).
- 코드 스타일: 도메인 용어는 한글 주석 병기, 함수/변수명은 영문 스네이크케이스.

## 성능 목표

- 1만 명 명부 기준 수 초 내 완료. 개인별 루프는 허용하되 내부 연도 루프는 numpy 벡터화.
